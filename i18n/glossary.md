# Glossary

- NeonCore Atlas: the product name. Do not translate.
- profile: a saved configuration group containing subscriptions, nodes, and routing settings.
- node: a server entry users can connect through.
- subscription: a remote link that provides profile or node data.
- routing: rules deciding how traffic is handled.
- global mode: route all traffic through the selected node.
- rule mode: route traffic based on rules.
- direct mode: do not use a proxy.
- connect: start a connection.
- disconnect: stop a connection.
- latency: measured delay to a node.
- tunnel: protected traffic path managed by the platform VPN layer.
- daemon: background NeonCore process on desktop platforms.
- service: OS-managed background component.
- proxy: network intermediary.
- VPN: platform virtual private network capability.
- log: recorded diagnostic event.
- diagnostics: information used to understand app or network state.
