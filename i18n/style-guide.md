# Localization Style Guide

- Keep UI text short and clear.
- Avoid slang.
- Avoid unnecessary technical jargon in basic UI.
- Keep complete sentences together.
- Do not split sentences into fragments for concatenation.
- Use placeholders for dynamic values.
- Add translator context for ambiguous strings.
- Prefer "node" for server entries unless a platform-specific UI has a better label.
- Prefer "profile" for imported or manually created configuration groups.
- Prefer "subscription" for remote subscription links.
